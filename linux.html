File Permissions In Linux

The focus on this project is changing file and directory permissions for users in the Linux command line.

Check file and directory details

The first command is changing directory with cd so that I am working from the projects directory. Then I used ls -la to list the files and their permissions from this directory including any hidden files shown with a period at the beginning of the file name.

Describe the permissions string

For this example I will choose the file project_k.txt Currently the permissions (seen in line 8) are set to: User: read, write
Group: read, write
Other: read
The intended permissions should be:
User: read, write
Group: read
Other: none

Change file permissions

I used the chmod command to set the file permissions to the specified group=read only and other=none. Then I used ls -la to reprint after executing the previous command to show the updated permissions.

Change file permissions on a hidden file

For this example, I am working with the hidden file .project_x.txt
The permissions needed to be updated from user:read to no permissions on this particular file.
I have used chmod again, and then ls -la to reprint the permissions after the changes.

Change directory permissions

This example is to update the permissions for the drafts directory so that group has read permissions only. I have used chmod and specified the directory of drafts so that it applies to the entire directory.

Summary

I have shown the ability to navigate directories in the Linux command line and to change file and directory permissions, including hidden files.
I have used the following commands:
cd - change directory
ls -la - list permissions of files, including hidden files
chmod - change permissions of files and directories
